# Teammate 3 — Work Breakdown

## Step 1 — Admin Layout + Sidebar + Routing + Auth Guard (Day 1)
Admin layout shell, sidebar nav (10 modules), breadcrumbs, responsive, admin auth guard middleware, mock admin session.

## Step 2 — Admin Dashboard (Day 2)
KPI cards (total users, active listings, active rentals, pending disputes, pending payouts), revenue chart (daily/weekly/monthly), order status funnel, recent activity feed.

## Step 3 — User Management + Product Listing Management (Day 3)
User table with search/filter/status, user detail drawer, suspend/unsuspend/delete actions. Product listing table, approve/reject/suspend, category management CRUD.

## Step 4 — Order Management + Delivery Management (Day 4)
Order table with filters (status, date, delivery type, dispute flag), order detail with photo viewer, force-cancel, reassign agent. Delivery orders table, agent assignment, agent performance view.

## Step 5 — Dispute Management + Payout Management (Day 5)
Dispute queue sorted by days pending, side-by-side photo comparison (pickup vs return), deduction input, resolve action. Payout queue with hold countdown, manual release, block payout, payout history with filters.

## Step 6 — Financial Reports + Platform Configuration (Day 6)
Reports: GMV, platform fees, deposits, late fees, delivery fees, refunds — by period. CSV/PDF export. Platform Config: all configurable params from PRD 4.3.9 as form fields (Super Admin only).

## Step 7 — Role & Permission Management + Final Testing (Day 7)
Sub-admin CRUD, permission toggles per module (read/write/delete), audit log viewer. Final a11y testing, responsive testing, all modules tested with mock data.
